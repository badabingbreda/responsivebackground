Advanced Demo

Depending on your needs you might need diffent @media queries, resizing your masonry grid with applied background. But on mobile you might only need very small images, while for bigger screens you need bigger images. The responsiveBackground will always attempt to return the smallest possible image to only download what's needed.

In this example we create 200 grid-item elements using javascript and alternate the backgrounds between a kitty and a hitchhiker. For each image a <div> element is created with a data-srcset and data-src property. We can re-use the div for each set to minimize the code. We use a <div> element because as opposed to <img> element the src-property will have no initial load an image. in the options of responsiveBackground we set the img-parameter, to use that div.cat_images element.

Now resize and watch what happens!



	//set a responsive Background for each odd grid-item
	$('.grid-item:nth-child(odd)').responsiveBackground( { img: '.cat_images' } );

	//set a responsive Background for each even grid-item
	$('.grid-item:nth-child(even)').responsiveBackground( { img: '.atlas_images' } );
