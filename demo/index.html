Demo 2

Hero image using first <img> found under element. This time, breakpoints were set. When under a certain 'width', use smallest possible image that will at least fit 'imagesize'. When setting the fallbackbreakpoint any window size outside the range of the breakpoints an image of at least 'imagesize' (but nearest) will be used.

In this example the background is at very low quality when viewed on small screens, at highest quality from 350-800 pixels and on medium quality for bigger screens. This way, when serving a masonry grid, you don't need to compromise on which imageresolution to use.



		$('.hero-image.demo2').responsiveBackground({
			breakpoints: 	[
				{width:350, imagesize: 100},
				{width:800, imagesize: -1}
			],
			fallbackbreakpoint: { width: 1024, imagesize: 400 }
		});

Demo 3

Hero image using <img> found using a specified element, here using the .demo1 img.

If needed you can reset the source image. In this example when we click on the Hero header it is set to the img used in 'Demo 2'. We need to call the 'redraw' method to make sure it is applied immediately.




		$('.hero-image.demo3').responsiveBackground({
			img: '.demo1 img',
			breakpoints:[
					{ width: 300, imagesize: 500 },
					{ width: 800, imagesize: 300 }
				],
			fallbackbreakpoint: { width: 1024, imagesize: -1 }
			});

		$('.hero-image.demo3')
			.on( 'click' ,
				function() {
					$(this).responsiveBackground( 'setImg', '.demo2 img' );
					$(this).responsiveBackground( 'redraw' );
				});
